# AI 伦理检查清单（AirRadar 优化 Demo）

## 1. 数据与使用范围

- [ ] 数据来源合法且注明（公开空气质量与气象数据）
- [ ] 数据处理流程留档（清洗、标准化、划分、mask 节点策略）
- [ ] 无个人敏感信息与可识别个体轨迹
- [ ] 地理位置可能存在再识别风险的评估（需要时网格/模糊处理）

## 2. 公平性（Fairness）

- [ ] 分区域/密度的误差对比报表
- [ ] 低密度区域系统性偏差阈值定义与监控
- [ ] 支持权重/再采样策略选项（当前占位）
- [ ] 公平性指标随版本迭代记录

## 3. 透明性与可解释性

- [ ] 模型版本 + 训练配置自动保存
- [ ] 频域能量分布输出（占位）
- [ ] 局部注意力权重可视化接口（待实现）
- [ ] 重要特征影响度或敏感度分析（后续可添加）

## 4. 能源与环境影响

- [ ] 训练能耗估算函数（样本数×epoch×设备系数）
- [ ] 建议混合精度与频率裁剪选项（后续）
- [ ] 蒸馏或轻量推理模型计划（后续）

## 5. 安全与责任

- [ ] 输出结果用途声明（仅辅助，不替代官方监测）
- [ ] 不确定性/风险提示（后续）
- [ ] 错误用例（极端天气、传感器故障）测试计划

## 6. 维护与更新治理

- [ ] 每次数据集更新触发再训练与公平性回归测试
- [ ] 过期模型自动下架流程（后续）
- [ ] 伦理评估结果纳入发布审批

## 7. 风险缓解与应急预案

- [ ] 模型异常（输出突变）自动告警
- [ ] 迅速回退到上一稳定版本的机制
- [ ] 与专家的复核接口预留

## 8. 合规与文档

- [ ] README 中说明用途与局限
- [ ] 伦理清单与评估脚本开源
- [ ] 版本变更日志包含伦理相关变更
